# v0.10.0 — Stdlib Consolidation (BREAKING)

## Why

Six modules in the stdlib overlapped and three names collided with
common Go std-package FFI bindings:

| Concern | Old (duplicates) | New (canonical) |
|---|---|---|
| CLI args | `Args.getArgs`, `Args.getArg`, `System.args` | `System.{args, getArg}` |
| Env vars | `System.getenv`, `Process.getEnv`, `Env.{get, getOrDefault, require, getInt, getBool}` | `System.{getenv, getenvOr, getenvInt, getenvBool}` |
| Cwd | `System.cwd`, `Process.getCwd` | `System.cwd` |
| Process exit | `System.exit`, `Process.exit` | `System.exit` |
| SHA-256 | `Crypto.sha256`, `Sha256.sum256`, `Sha256.sum256String` | `Crypto.sha256` |
| Hex | `Encoding.hexEncode/Decode`, `Hex.encode/encodeToString/decode` | `Encoding.hexEncode/Decode` |
| Logging | `Log.{info,warn,…}`, `Slog.{info,warn,…}` | `Log.*` (plain) + `Log.*With` (structured) |

Plus the Sky kernel `Os` module owned the bare `Os` qualifier, which
broke `import Os` for users who needed the **Go** `os` package
(stdin / stderr / fileWriteString — sky-log's case).

## What changed

### Renamed
- `Os` → `System` for the Sky kernel. Use `System.exit`, `System.getenv`,
  `System.cwd`, `System.args` going forward. The `Os` qualifier is now
  free for the Go FFI binding.

### Dropped (folded into other modules)
- **`Args.*`** → `System.{args, getArg}`. `Args.getArgs ()` is now
  `System.args ()`. `Args.getArg n` is `System.getArg n` (returns
  `Task Error (Maybe String)` per Task-everywhere).
- **`Env.*`** → `System.{getenv, getenvOr, getenvInt, getenvBool}`.
  `Env.getOrDefault key def` → `System.getenvOr key def`.
  `Env.getInt key` → `System.getenvInt key`. `Env.require key` →
  `System.getenv key` (already errors on missing).
- **`Slog.*`** → `Log.*With`. `Slog.info msg attrs` →
  `Log.infoWith msg attrs` (same for warn/error/debug). Plain
  `Log.info msg` keeps working — the `With`-suffix variants take the
  structured `(msg, [k,v,k,v,…])` shape Slog had.
- **`Sha256.*`** → `Crypto.sha256`. The chain
  `Sha256.sum256 (String.toBytes s) |> Result.andThen Hex.encodeToString`
  collapses to `Crypto.sha256 s` (returns the hex digest directly).
- **`Hex.*`** → `Encoding.hexEncode` / `Encoding.hexDecode`.

### Shrunk
- **`Process.*`** keeps only `run` (subprocess execution). `exit` /
  `getEnv` / `getCwd` / `loadEnv` all moved to `System.*`.

### Added
- `System.getenvOr key default` — never errs, returns default if unset.
- `System.getenvInt key` / `System.getenvBool key` — typed env reads.
- `System.getArg n` — single-arg lookup as `Task Error (Maybe String)`.
- `System.loadEnv ()` — load `.env` file (was `Process.loadEnv`).
- `Log.infoWith msg attrs` / `warnWith` / `errorWith` / `debugWith` —
  structured-log variants with key/value list (replaces `Slog.*`).
- sky.toml `[log] format = "plain" | "json"` and
  `[log] level = "debug" | "info" | "warn" | "error"`. Seeds
  `SKY_LOG_FORMAT` / `SKY_LOG_LEVEL` defaults at compile time. Env vars
  still override at runtime — three-layer precedence (env > `.env` >
  `sky.toml`) matches `[live]`.

### Compiler infrastructure
- Bare-name aliases for every kernel module added to
  `staticKernelModules`. `Log.error` resolves without
  `import Std.Log` because `Log` is in the kernel registry.
- `main`'s body is wrapped in `rt.AnyTaskRun` — `main = println X`
  actually prints under Task-everywhere (regression discovered when
  migrating examples).
- Canonicaliser falls back to the kernel registry for unimported
  qualifiers — `Crypto.sha256` works without `import Sky.Core.Crypto`.
  Bug fixed: previously emitted bare `Crypto_sha256(arg)` (no `rt.`
  prefix), failing `go build`.
- `Basics.identity` emits `[any]` generic instantiation when used as
  a value (e.g. `List.filterMap identity [...]`), unblocking sky-log.

## Migration

For Sky projects, the bulk rewrites can be sed-applied:

```bash
# Slog → Log.*With
perl -i -pe 's/\bSlog\.(info|warn|error|debug)\b/Log.${1}With/g' src/**/*.sky

# Args → System
perl -i -pe 's/\bArgs\.getArg\b/System.getArg/g; s/\bArgs\.getArgs\b/System.args/g' src/**/*.sky

# Env → System
perl -i -pe 's/\bEnv\.getOrDefault\b/System.getenvOr/g; s/\bEnv\.getInt\b/System.getenvInt/g; s/\bEnv\.getBool\b/System.getenvBool/g; s/\bEnv\.require\b/System.getenv/g; s/\bEnv\.get\b/System.getenv/g' src/**/*.sky

# Process.{exit,getEnv,getCwd,loadEnv} → System.*
perl -i -pe 's/\bProcess\.exit\b/System.exit/g; s/\bProcess\.getEnv\b/System.getenv/g; s/\bProcess\.getCwd\b/System.cwd/g; s/\bProcess\.loadEnv\b/System.loadEnv/g' src/**/*.sky

# Os → System for kernel calls
perl -i -pe 's/\bOs\.(exit|getenv|cwd|args)\b/System.$1/g' src/**/*.sky

# Imports
perl -i -pe 's/^import Sky\.Core\.Os as Os\n/import Sky.Core.System as System\n/g; s/^import Sky\.Core\.Args as Args\n/import Sky.Core.System as System\n/g; s/^import Std\.Env as Env\n/import Sky.Core.System as System\n/g; s/^import Log\.Slog as Slog\n/import Std.Log as Log\n/g; s/^import Crypto\.Sha256 as Sha256\n//g; s/^import Encoding\.Hex as Hex\n//g' src/**/*.sky
```

Manual touch-ups needed for:
- `case Args.getArg n of Just/Nothing` patterns (now wrap with
  `Task.run … |> Result.withDefault Nothing`).
- `Sha256.sum256 + Hex.encodeToString` chains (collapse to
  `Crypto.sha256`).
- `case Args.getArgs () of …` (now Task-typed; collapse with
  `Task.run … |> Result.withDefault []`).
